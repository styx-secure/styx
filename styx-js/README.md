# Styx.js — Sovereign P2P Cryptographic Ledger for the Browser

JavaScript port of the Styx Dart/Flutter library. Two peers maintain a shared, tamper-evident event chain with E2E encryption — no central server stores any data.

**Fully tested:** 431 Jest tests + 17 Playwright browser tests + 22 cross-platform interop tests against Dart.

## Architecture

```
┌─────────────────────────────────────────────────┐
│  5. Facade (SovereignLedger)                     │  Pairing, re-keying, backup
├─────────────────────────────────────────────────┤
│  4. Transport Layer                              │  Nostr (primary) + WebRTC (optional)
├─────────────────────────────────────────────────┤
│  3. Ledger Engine                                │  Hash chain, vector clocks, merge
├─────────────────────────────────────────────────┤
│  2. Storage Layer                                │  Memory (ephemeral) / IndexedDB
├─────────────────────────────────────────────────┤
│  1. Crypto Layer                                 │  Ed25519, SPAKE2, Shamir, ChaCha20
└─────────────────────────────────────────────────┘
```

## Quick Start

```js
import {
  SovereignLedger, LedgerConfig,
  MemoryLedgerStore, MemoryPeerStore, MemoryKeyStore, MemoryOutboxStore,
} from 'styx-js';

const ledger = new SovereignLedger({
  config: new LedgerConfig({
    relayUrls: ['wss://nos.lol', 'wss://relay.snort.social'],
  }),
  ledgerStore: new MemoryLedgerStore(),
  peerStore:   new MemoryPeerStore(),
  keyStore:    new MemoryKeyStore(),
  outboxStore: new MemoryOutboxStore(),
});

await ledger.initialize();
// State: 'unpaired'
```

### QR (local)

```js
// Device A generates QR data
const qrData = await ledger.generatePairingQr();
// → JSON: {"pk":"hex...","n":"base64...","r":["wss://..."]}

// Device B processes scanned QR

  await ledger.confirmPairing({
    peerPublicKey: result.peerPublicKey,
    peerAlias: 'Alice',
  });

```

### Remote (mnemonic + SPAKE2 + Double Check)

```js
// Device A — initiator
const mnemonic = await ledger.startRemotePairing();
// → "abandon ability able about above absent"
// Share this verbally with Device B

// Device B — responder
await ledger.startRemotePairing('abandon ability able about above absent');

// Both devices derive the same 6-digit verification code
const code = await ledger.getDoubleCheckCode();
// → "483 291"

// After verbal confirmation

// State: 'ready'
```

## Sending & Receiving Events

```js
await ledger.sendMessage({
  payload: new TextEncoder().encode(JSON.stringify({
    type: 'text', text: 'Hello from Styx!',
  })),
});

ledger.eventStream.onRemoteEvents((event) => {
  const data = JSON.parse(new TextDecoder().decode(event.payload));
  console.log('Received:', data);
});
```

## Privacy Model

| Transport | Peer sees your IP? | Relay sees your IP? | Relay reads content? |
|-----------|:------------------:|:-------------------:|:--------------------:|
| **Nostr relay** | No | Yes | No (E2E encrypted) |
| **Nostr + Tor** | No | No | No |
| **WebRTC direct** | Yes | N/A | N/A |
| **WebRTC via TURN** | No | TURN sees IP | No |

Styx uses **Nostr as the primary transport** — messages are encrypted with ChaCha20-Poly1305 before leaving the browser. The relay sees only ciphertext. Peers never see each other's IP address.

WebRTC is an optional low-latency channel where both peers accept the IP visibility tradeoff.

## Storage Modes

| Mode | Store Classes | Persistence | Use Case |
|------|-------------|-------------|----------|
| Ephemeral | `Memory*Store` | Tab lifetime | Disposable chats |
| Persistent | `IndexedDB*Store` | Browser storage | Long-term ledgers |

## Transport Hierarchy

1. **Nostr relays** — Primary channel, E2E encrypted, no IP leak to peer
2. **WebRTC DataChannel** — Optional direct P2P for low-latency use cases

Events are signed with NIP-01 schnorr/secp256k1 for relay acceptance. Content is encrypted with ChaCha20-Poly1305 (12-byte nonce, compatible with Dart implementation).

## Dependencies

- `@noble/ed25519` — Ed25519 signatures
- `@noble/curves` — P-256 (SPAKE2), X25519 (DH), secp256k1 (Nostr NIP-01)
- `@noble/hashes` — SHA-256, SHA-512, HKDF, HMAC, PBKDF2
- `@noble/ciphers` — ChaCha20-Poly1305

All `@noble` libraries are audited, zero-dependency, and pure JS.
