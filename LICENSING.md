# Licensing map — styx-secure/styx

This document is the authoritative, human-readable licensing map of this
repository. The machine-readable equivalent is [`REUSE.toml`](REUSE.toml)
(REUSE Specification 3.3). The model was approved by the copyright holder,
Maurizio Verde, in GitHub Issue #40 and implemented under GitHub Issue #41.

## License precedence

Licensing is resolved per exact path, in this closed order of precedence:

1. **Exact third-party and derivative classifications** (listed below).
   A third-party or derivative classification always wins over any Styx
   default; the AGPL default never overwrites it.
2. **Exact human-approved Apache-2.0 vector exceptions** (the six files
   listed below, and nothing else).
3. **`AGPL-3.0-or-later` default** for all remaining original Styx material.

No broad directory glob grants Apache-2.0. Any future Apache-2.0 exception
requires a separate inventory, an exact path list, an explicit amendment by
the copyright holder and an independent licensing review.

## 1. Default: AGPL-3.0-or-later

Unless a classification below applies, every file in this repository is:

```text
SPDX-License-Identifier: AGPL-3.0-or-later
SPDX-FileCopyrightText: 2026 Maurizio Verde
```

This covers, among other things: all product source code (Dart, JavaScript,
Rust, Go), applications and services, Styx-authored tests and fixtures,

(including security and architecture documentation), and build/verification
tooling. The canonical license text is in [`LICENSE`](LICENSE), byte-identical
to [`LICENSES/AGPL-3.0-or-later.txt`](LICENSES/AGPL-3.0-or-later.txt).

Documentation is not dual-licensed and no Creative Commons license is used.

### styx-kdf-wasm (`styx-js/vendor/styx-kdf-wasm/`)

The crate is **original Styx software** (no external clone; it is built from
the committed sources only) and is licensed `AGPL-3.0-or-later`, as declared
in its `Cargo.toml`. Its `deny.toml` dependency-license policy is unchanged
in substance: the permissive allowlist still constrains every dependency;
the unpublished root crate itself is excluded from that check
(`private = { ignore = true }`) instead of adding AGPL to the allowlist.

Generated outputs under `pkg/`:

- `styx_kdf_wasm.js` — generated from the AGPL crate; embeds wasm-bindgen MIT
  runtime portions → `AGPL-3.0-or-later AND MIT`;
- `styx_kdf_wasm_bg.wasm` — compiled aggregate of the AGPL crate plus
  statically linked permissive crates (RustCrypto stack under its MIT
  option, `generic-array` MIT, `subtle` BSD-3-Clause) →
  `AGPL-3.0-or-later AND MIT AND BSD-3-Clause`;
- the TypeScript declarations and `SHA256SUMS` fall under the AGPL default.

The same layering applies to the committed Argon2id spike artifact under
`styx-js/spikes/argon2id/crate/pkg/`.

### Generated files and package-manager metadata

Generated files are mapped to the license of their source or relevant
derived work, as listed above. Lockfiles and other package-manager metadata
committed by Styx (`package-lock.json`, `Cargo.lock` outside the OpenMLS
vendor directory, `pubspec.yaml` workspace files) fall under the AGPL
default; they carry no independent creative third-party authorship. The
OpenMLS vendor `Cargo.lock` is classified upstream MIT as noted above.

Dependencies fetched at build time from npm, crates.io or pub.dev are not
contained in this repository and keep their own licenses; they are not part
of this map.

## Package metadata

- `styx-js/package.json` declares `"license": "AGPL-3.0-or-later"` and
  `"private": true`. The root entry of `styx-js/package-lock.json` is
  synchronized with it.
- `styx-js/vendor/styx-kdf-wasm/Cargo.toml` declares
  `license = "AGPL-3.0-or-later"` with `publish = false`.
- Nested private packages without a `license` field
  (`styx-js/apps/chat`, `push_bridge`, `styx-js/spikes/argon2id`) are
  deliberately not modified; their licensing is expressed through
  `REUSE.toml`.
- Dart `pubspec.yaml` files have no license field (the format does not
  support one); every Dart package is `publish_to: none` and is covered by
  the AGPL default.

## Root NOTICE decision

No root `NOTICE` file exists, deliberately. A `NOTICE` file in the sense of
Apache License 2.0 §4(d) must be preserved only when upstream Apache-licensed
material ships one. The inventory (Issue #41, Phase A) found none: OpenMLS is
MIT with no NOTICE file at the pinned commit, and the dual-licensed
`MIT OR Apache-2.0` Rust crates embedded in the committed artifacts are
distributed here under their MIT option. Third-party attribution lives in
[`THIRD_PARTY_NOTICES.md`](THIRD_PARTY_NOTICES.md); `NOTICE` is not used as a
generic dependency list. If Apache-licensed material with a NOTICE file is
ever vendored, that NOTICE must be reproduced verbatim.
