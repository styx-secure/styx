# Third-party notices — styx-secure/styx

This file inventories the third-party, derived and vendored material
**contained in this repository**, with the attribution required by the
respective licenses. It complements [`LICENSING.md`](LICENSING.md) and
`REUSE.toml`; canonical license texts are in [`LICENSES/`](LICENSES/).

Dependencies that are only *fetched* at build or install time (npm,
crates.io, pub.dev, Go modules) are not repository content and are not
listed here; their licenses travel with the packages themselves.

Nothing in this file claims Styx authorship of upstream code, and nothing in
it reclassifies upstream code under a Styx license.

## 2. Rust crates statically linked into the committed WASM artifacts

The repository commits three compiled WebAssembly artifacts. Each is an
aggregate that statically links third-party Rust crates. The dependency
graphs are pinned by the respective committed `Cargo.lock` files, and for
`styx-kdf-wasm` they are additionally enforced by `deny.toml`
(`cargo deny`) and documented in its `PROVENANCE.md`.

Artifacts:

1. `styx-js/vendor/openmls-wasm/openmls_wasm_bg.wasm`
2. `styx-js/vendor/styx-kdf-wasm/pkg/styx_kdf_wasm_bg.wasm`
3. `styx-js/spikes/argon2id/crate/pkg/argon2id_spike_bg.wasm` (prototype)

Notes:

- For every crate licensed `MIT OR Apache-2.0`, this repository elects the
  **MIT** branch for the committed artifacts. As a consequence no Apache
  §4(d) NOTICE-preservation obligation arises from these crates.
- Build-time-only dependencies (proc-macros such as `syn`, `quote`,
  `wasm-bindgen-macro`, `unicode-ident`) do not reach the compiled
  artifacts and are listed only in the lockfiles.
- The OpenMLS workspace lockfile pins additional crates (e.g. `libcrux-*`,
  `ring`, `openmls-fuzz`) that belong to workspace members **not** compiled
  into artifact 1 (the build uses the RustCrypto provider); they are not
  repository-contained code.
