# Contributing to Styx

Thank you for your interest in Styx. Please read this page before opening a
pull request: the contribution process is intentionally restricted at this
stage.

## External code contributions are temporarily paused

The project is **not accepting external code contributions** (pull requests
with source, test, build or documentation-content changes from outside the
project) at this time. Contributor terms capable of sustaining the project's
licensing model are still being defined; defining them is a **separate,
future task** with its own human approval.

Concretely, until that task lands:

- external code pull requests will be closed without merge, regardless of
  quality — please do not invest implementation effort yet;
- **no CLA exists and none is being introduced by this repository today**;
  there is no copyright assignment and no DCO requirement in force;
- when contributor terms are approved, this file will be updated first.

## What is welcome right now

- **Issues**: bug reports, reproducible problems, usability feedback and
  well-scoped feature discussion are welcome through
  [GitHub Issues](https://github.com/styx-secure/styx/issues).
- **Documentation feedback**: unclear, wrong or missing documentation is a
  bug — please report it as an Issue (as feedback, rather than as a content
  pull request, while contributions are paused).

## Security vulnerabilities

**Do not open a public Issue for a vulnerability.** Use GitHub's private
vulnerability reporting for this repository (Security tab → *Report a
vulnerability*), which is enabled and is the approved private reporting
channel. You should receive an acknowledgement through that channel; please
allow reasonable time for triage before any disclosure.
